## Getting started 
Before starating, there a few things you need to do. 
1. Clone/Fork the repo and check you have the right libaries: 
``` bash 
git clone https://github.com/KnotFaries/CS375_Semester_Project.io.git
```

Also make sure you have python and the following libraires: souddivece, tkinter, numpy, time, sys
## Before you commit 
Set up a linter. Currently we use PEP 8 as a style guide. If you are using an andcoda IDE you should be fine. 
## Branching
- 'main' is alwasy deployable. Never comit directly to main. 
-Branch name: clear and concise
    - When working with features or fixes use 'feature/description'
    - When working with documetaiong use 'doc/name_of_doc/date_range'
## Pull Request
- Always go through a pull request. Never comit to main. 
- PR title should be clear on what has changed, as well as how to test or any work that must be done after the PR. 
- Get aproval from somone esle before merging
